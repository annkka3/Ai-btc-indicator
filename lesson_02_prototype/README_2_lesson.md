# AI Индикатор рынка BTC — Прототип продукта


## 1) Какую задачу решает продукт?

Помочь частным и полупрофессиональным трейдерам **быстро получить объяснимый краткосрочный прогноз по BTC** (1ч/4ч/24ч) и принять решение в контексте режима рынка (волатильность, позиционирование, риски) **прямо в Telegram**.

**Боль:**

- Сигналки шумные и непрозрачные.

- TradingView-индикаторы статичны и не дают вероятностей.

- Аналитические сайты — тяжёлые и не про «быстро».

**Решение:**

- **Telegram-карточка прогноза** с вероятностью направления, величиной отклонения от MA\_L (EMA/KAMA) и ключевыми факторами (explainability).

- Доставка ≤ 30–120 с после закрытия бара (**только финальные бары**).

- Объяснимые прогнозы с контекстом рынка (режим/волатильность/ликвидность).

## 3) Какую ценность продукт приносит ЦА? (и как измерим)

### Ценность

- **Мгновенная объяснимая карточка прогноза** (1ч/4ч/24ч) в Telegram.

- **Уменьшение шума**: минимально-достаточный набор метрик с понятной шкалой.

- **Экономия времени**: не нужно собирать данные с разных сайтов.

- **Контекст рынка**: режим (волатильность/тренд/ликвидность) для интерпретации.

- **Последующая персонализация**: по мере развития проекта превращение бота в персрнализованного помощника по контролю рисков над портфелем 


### Как измерим (product KPIs)

- **Activation**: % новых, сделавших ≥ 1 запрос в первые 24ч.

- **Engagement**: карточек/польз./день; доля «quick open < 60с».

- **Retention**: D7/D30 по событию «получил прогноз».

- **NPS/CSAT** внутри бота.

- **Conversion to paid**

### Как измерим (ML-вклад)

- Доля прогнозов, прошедших **Release Gate** 

- Калибровка вероятностей (Brier/ECE) и стабильность по режимам рынка.

## 9) MVP-объём и что вне скопа

**Входит (MVP):** 1ч/4ч/24ч карточки; residual-таргет (лог-отклонение от MA\_L); 8–12 базовых фич; авто- и on-demand доставка; шкалы/легенда; NPS/CSAT; freemium.  

**В бэклог:** опционы (GEX/vanna/charm), ончейн-агрегаты (NUPL/MVRV/SOPR), ордербук L1–L10 (imbalance/OFI), ETF-AUM детализация, расширенные корреляции/бета, портфели, альткоины.

---

## 10) Метрики (product ↔ ML ↔ ops)

**Product:** Activation, Engagement, Retention (D7/D30), Conversion-to-paid, NPS/CSAT.  

**ML:** MAE-gain (vs нулевой прогноз) ≥ 5%/6%/8% (1ч/4ч/24ч), AUC ≥ 0.58/0.60/0.62, Sharpe (policy long-if-ŷ>0) ≥ 1.0/1.1/1.2; Brier/ECE (калибровка); стабильность по режимам.  

**Ops:** свежесть ≤ 30–120с; p95 инференса ≤ 2с; p95 доставки ≤ 5с; ≤ 1% 4xx/5xx/timeout в сутки; 100% карточек с версией/хэшем.

**Release Gate (per-H):** релиз, если выполнены **2 из 3**: {MAE-gain, AUC, Sharpe} на OOS.  

**No-Go:** иначе — перенос релиза и доработка (фичи/калибровка/режимный гейт).

## 11) План быстрых экспериментов

- **А/В контента карточки** (краткая vs подробная) → open/click, CSAT.

- **Шкалы/цветовые бейджи** для P(up)/режима → понимание (опрос).

- **Частота авто-доставки** (каждый close vs окна) → engagement.

- **Порог уведомлений** (|Δ vs MA| > X) → удержание/отписки.
